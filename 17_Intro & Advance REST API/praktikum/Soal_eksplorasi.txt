1. Jelaskan perbedaan antara pengembangan aplikasi mobile secara hybrid (contoh: Flutter) dan pengembangan aplikasi mobile secara native (contoh: Kotlin untuk pengembangan aplikasi Android)
Jawaban
A. Pendekatan dan Bahasa Pemrograman:
Hybrid (Flutter): Pendekatan hybrid menggunakan kerangka kerja lintas platform seperti Flutter, yang memungkinkan pengembang untuk menulis satu kode yang dapat dijalankan di berbagai platform. Dalam kasus Flutter, bahasa pemrograman yang digunakan adalah Dart.
Native (Kotlin): Pendekatan native memanfaatkan bahasa pemrograman dan alat pengembangan yang spesifik untuk setiap platform. Contoh ini menggunakan Kotlin, bahasa pemrograman yang resmi didukung oleh Google untuk pengembangan aplikasi Android.
B. Kinerja:
Hybrid (Flutter): Meskipun Flutter berusaha untuk mendekati performa aplikasi native, beberapa aplikasi mungkin mengalami sedikit penurunan kinerja dibandingkan dengan aplikasi native karena adanya lapisan abstraksi.
Native (Kotlin): Pengembangan native cenderung memberikan kinerja terbaik karena aplikasi dikompilasi langsung ke kode mesin platform target.
C. UI/UX:
Hybrid (Flutter): Flutter menawarkan kerangka kerja UI yang kaya dengan elemen-elemen yang sepenuhnya dapat disesuaikan, tetapi mungkin memerlukan penyesuaian lebih lanjut untuk merespons perilaku platform yang unik.
Native (Kotlin): Pengembangan native memungkinkan akses langsung ke API UI platform, sehingga memberikan pengalaman yang lebih konsisten dengan pedoman desain masing-masing platform
D.Akses ke Fitur Platform:
Hybrid (Flutter): Dalam beberapa kasus, akses ke fitur platform mungkin memerlukan plugin khusus atau kode tambahan untuk berinteraksi dengan API platform tertentu.
Native (Kotlin): Pengembangan native memungkinkan akses langsung ke seluruh fitur dan API platform tanpa perlu lapisan abstraksi.
E. Pengembangan Cepat:
Hybrid (Flutter): Pengembangan dengan kerangka kerja lintas platform seperti Flutter dapat memungkinkan pengembangan lebih cepat karena satu kode dapat digunakan di berbagai platform.
Native (Kotlin): Pengembangan native mungkin memerlukan lebih banyak waktu dan upaya karena perlu mengembangkan kode terpisah untuk setiap platform.

2. Sebutkan contoh kasus yang tepat untuk pengembangan aplikasi mobile secara hybrid.
Contoh Kasus Tepat untuk Pengembangan Aplikasi Mobile Secara Hybrid:
Jawaban
A. Aplikasi Berita dan Konten Media adalah Aplikasi yang menghadirkan berita, artikel, video, atau konten media lainnya dapat dikembangkan secara hybrid.
B. Aplikasi Bisnis Internal adalah Untuk perusahaan yang membutuhkan aplikasi bisnis internal, seperti alat manajemen inventaris atau alat pelaporan internal, pengembangan hybrid dapat menjadi pilihan yang baik.
C. Aplikasi Katalog Produk adalah Jika Anda ingin membuat aplikasi katalog produk atau toko online sederhana, pengembangan hybrid dapat memberikan solusi yang efektif, terutama jika tujuannya adalah untuk memiliki kehadiran di berbagai platform dengan cepat.
D. Aplikasi E-commerce adalah Aplikasi e-commerce yang lebih sederhana dengan fokus pada navigasi produk, pengecekan harga, dan transaksi pembayaran dapat dikembangkan secara hybrid untuk menghemat waktu dan sumber daya.
E. Aplikasi yang Tidak Memerlukan Fungsionalitas Khusus Platform: Jika aplikasi Anda tidak memerlukan fungsionalitas atau integrasi khusus dari platform, pengembangan hybrid dapat menjadi pilihan yang efisien.
